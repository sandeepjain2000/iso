Purpose
-------
Design, compliance, and evaluation materials (ISO / US compliance framing), prototypes (screens_prototype.html), screenshots, and related archives.

How to use
----------
1. Open HTML prototypes locally in a browser; treat DOCX and ZIP as source documents for audits or product design reviews.
2. Use US_COMPLIANCE and system/ folders as reference libraries when writing policies or assessment prompts.
3. Screenshots are evidence or UX references; rename with dates when adding new captures.
4. Do not treat this folder as executable software unless you add runnable code with its own instructions.
